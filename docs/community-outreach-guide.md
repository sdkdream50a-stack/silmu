## 커뮤니티별 접근 전략

### 네이버 카페 전략 (가장 중요)

#### Phase 1: 신뢰 구축 (1~2주)

1. **프로필 설정**
   - 닉네임: "계약실무가이드" 또는 "법령도우미"
   - 자기소개: 간결하게 "공무원 실무 도움 드립니다"
   - 실무.kr 링크는 프로필에만 (게시글 X)

2. **초기 활동**
   - 질문 답변만 (링크 없이)
   - 10~15개 답변 작성
   - 다른 회원들의 신뢰 획득

#### Phase 2: 자연스러운 링크 공유 (2주~)

3. **링크 공유 시나리오**

**시나리오 A: 질문 답변**
```
Q: "수의계약 한도액이 몇천만원인가요?"

A: "물품·용역은 2천만원 이하입니다. 다만 특례가 있어서...
(자세한 설명)

참고로 여기에 수의계약 한도액 표가 정리되어 있어요:
https://silmu.kr/topics/private-contract-amount

도움이 되셨길 바랍니다!"
```

**시나리오 B: 감사사례 공유**
```
제목: [감사사례] 예정가격 누설 사건 - 파면·실형 처벌

최근 감사원에서 적발된 예정가격 누설 사건입니다.
담당 공무원이 입찰 업체에 예정가격을 알려줬다가 파면+실형 받았네요.

사건 전말과 교훈을 정리한 글이 있어서 공유합니다:
https://silmu.kr/audit-cases/estimated-price-leak-incident

다들 조심하세요!
```

**시나리오 C: 신규 콘텐츠 알림**
```
제목: [법령 개정] 2026년 낙찰하한율 변경 (87.745% → 89.745%)

낙찰하한율이 올해 1월부터 89.745%로 변경되었습니다.
기존 87.745%와 혼동하지 않도록 주의하세요.

자세한 내용:
https://silmu.kr/topics/lowest-bid-rate

실무에 도움이 되시길!
```

#### Phase 3: 지속적 관계 유지

4. **주간 활동 루틴**

## 콘텐츠 공유 가이드라인

### 공유 우선순위

#### Tier 1 (항상 공유)

1. **신규 토픽** (법령 가이드)
   - 예: 낙찰하한율, 비교견적서 수집 방법
   - 공유 시기: 발행 즉시

2. **감사사례** (실명 사건)
   - 예: 예정가격 누설, 계약보증금 미징구
   - 공유 시기: 발행 후 1주일 이내

3. **법령 개정 소식**
   - 예: 시행령 개정, 집행기준 변경
   - 공유 시기: 개정 즉시

#### Tier 2 (선택적 공유)

4. **자동화 도구 업데이트**
   - 예: 적격심사 자동 채점기, 예정가격 계산기
   - 공유 시기: 주요 기능 추가 시

5. **FAQ 업데이트**
   - 예: 새로운 질문 추가 시
   - 공유 시기: 월 1회 종합 정리

### 콘텐츠 형식

#### 제목 작성법

**좋은 예:**
- ✅ "[필독] 2026년 계약보증금 면제 기준 정리"
- ✅ "[감사사례] 부실 시공 은폐로 파면 처벌"
- ✅ "[법령 개정] 예산 편성 절차 변경 (2026.1.1 시행)"

**나쁜 예:**
- ❌ "실무.kr에 새 글 올렸어요!"
- ❌ "제 사이트 구경 와주세요"
- ❌ "최고의 공무원 실무 사이트"

#### 본문 작성법

**구조:**
1. **핵심 정보 요약** (2~3줄)
2. **상세 내용 일부** (5~7줄)
3. **링크** (자연스럽게)
4. **마무리 인사** (1줄)

**예시:**
```
[제목] 계약보증금 미징구로 8억원 손실 사건

□□군에서 계약보증금을 징구하지 않고 공사 계약을 체결했다가
업체 부도로 8억원 전액 손실이 발생했습니다.

계약담당자는 "업체 신용도가 높아 보여서 괜찮을 줄 알았다"고 했지만,
결국 파면 처분과 손해배상 책임까지 지게 되었네요.

사건 전말과 예방 체크리스트는 여기에 정리되어 있습니다:
https://silmu.kr/audit-cases/contract-guarantee-not-collected

다들 계약보증금 징구 절대 빼먹지 마세요!
```

#### 1. UTM 파라미터 사용

각 커뮤니티별로 고유한 UTM 파라미터를 붙여서 유입 추적:

```
https://silmu.kr/topics/private-contract-amount?utm_source=naver_cafe&utm_medium=community&utm_campaign=gongmuwon_dotcom
```

**UTM 파라미터 규칙:**
- `utm_source`: 커뮤니티 플랫폼 (naver_cafe, dcinside, blind 등)
- `utm_medium`: 항상 `community`
- `utm_campaign`: 구체적 커뮤니티명 (gongmuwon_dotcom, gyeyak_silmuja 등)

#### 2. GA4 커스텀 대시보드 설정

**주요 측정 지표:**
- 커뮤니티별 세션 수
- 페이지별 유입 경로
- 평균 세션 시간 (참여도)
- 이탈률

**대시보드 위치**: `/admin/analytics` (이미 구현됨)

### 효과 측정 주기

## 주의사항 및 금기

### ❌ 절대 하지 말아야 할 것

1. **스팸 행위**
   - 같은 커뮤니티에 하루 2개 이상 링크 게시
   - 모든 답변에 링크 첨부
   - 관련 없는 글에 링크 댓글

2. **과도한 자화자찬**
   - "최고의 사이트", "1등 가이드" 등
   - "제가 만든 사이트인데..." 식의 자랑
   - 다른 사이트 비하

3. **거짓 정보**
   - 법령 내용 왜곡
   - 감사사례 과장
   - 허위 통계

4. **커뮤니티 규칙 위반**
   - 각 커뮤니티의 홍보 정책 확인 필수
   - 관리자 경고 시 즉시 중단
   - 강제 퇴출 시 다른 계정으로 재진입 금지

### ⚠️ 주의해야 할 것

1. **톤앤매너**
   - 커뮤니티 분위기에 맞춰 조절
   - 디시인사이드: 가벼운 톤
   - 네이버 카페: 정중한 톤
   - Blind: 전문적 톤

2. **반응 모니터링**
   - 댓글 반응 확인 (긍정 vs 부정)
   - 부정 반응 시 즉시 대응 (사과, 삭제, 설명)
   - 논쟁 피하기

3. **법적 문제**
   - 저작권 침해 금지 (다른 사이트 콘텐츠 복사)
   - 명예훼손 주의 (감사사례 실명 언급 금지)
   - 개인정보 보호 (이메일, 전화번호 수집 금지)

## 실행 체크리스트

### Week 1 (신뢰 구축)

- [ ] 네이버 카페 3곳 가입 (공무원닷컴, 지방공무원 소통방, 계약 실무자 모임)
- [ ] 프로필 설정 (닉네임, 자기소개)
- [ ] 질문 답변 10~15개 작성 (링크 없음)

### Week 2 (자연스러운 링크 공유)

- [ ] 네이버 카페에 신규 토픽 1개 공유 (UTM 파라미터 적용)
- [ ] 디시인사이드에 감사사례 1개 공유
- [ ] 블로거 3명에게 협업 제안 메일 발송

### Week 3~4 (지속적 활동)

- [ ] 주간 루틴 정착 (월/수/금 활동)
- [ ] GA4 대시보드 확인 (유입 경로 분석)
- [ ] 백링크 5개 확보 (나무위키, 블로그 2곳, 카페 자료실 2곳)

### Month 2~3 (최적화)

- [ ] 유입률 높은 커뮤니티 집중 공략
- [ ] 효과 낮은 커뮤니티 전략 변경 또는 중단
- [ ] 새로운 커뮤니티 발굴 (공무원 유튜버 댓글 등)
