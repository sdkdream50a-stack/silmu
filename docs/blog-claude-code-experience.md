## 본문 (네이버 블로그용)

### 들어가며

저는 지방자치단체에서 일하는 50대 공무원입니다.
계약, 예산, 세무 — 매일 반복되는 실무를 하면서 늘 생각했습니다.

> "이걸 좀 더 편하게 할 수 없을까?"

수의계약 한도액이 얼마인지, 이번 달에 해야 할 업무가 뭔지, 계약 절차가 어떻게 되는지. 매번 법령집을 뒤지고, 선배에게 물어보고, 엑셀 파일을 찾아 헤맸습니다.

그래서 직접 만들기로 했습니다.
코딩을 전혀 모르는 상태에서.

---

### 클로드 코드를 만나다

2026년 2월 2일, 저는 **클로드 코드(Claude Code)**라는 도구를 처음 사용했습니다.

클로드 코드는 Anthropic이라는 AI 회사가 만든 **터미널 기반 AI 코딩 도구**입니다. 쉽게 말하면, 컴퓨터의 검은 화면(터미널)에서 AI와 대화하면서 코드를 만드는 도구입니다.

처음엔 반신반의했습니다. "코딩을 모르는 내가 이걸로 뭘 만들 수 있을까?"

하지만 한번 해보기로 했습니다.

---

### 8일 동안 만든 것들

2월 2일부터 2월 9일까지, 단 8일 만에 완성한 것들입니다.

**만들어진 결과물:**
- Ruby 파일 58개, 화면(뷰) 파일 65개
- 컨트롤러 26개, 서비스 로직 14개
- 총 커밋(저장 기록) 69회
- 하루 평균 8~9번의 코드 변경

비전공자인 제가 코드를 직접 작성한 건 거의 없습니다.
**AI에게 "이런 게 필요해"라고 말하면, AI가 코드를 만들어줬습니다.**

---

### 이렇게 대화했습니다

실제로 제가 AI에게 했던 말들입니다:

> "업무달력을 만들어줘. 공무원이 매월 해야 할 업무를 보여주는 거야."

> "업무를 클릭하면 AI가 업무처리 가이드를 자동으로 만들어주게 해줘."

> "서버 열어줘."

> "프로덕션 서버에도 API 키 설정해줘."

코드를 하나도 모르지만, **한국어로 원하는 것을 설명하면** AI가 알아서 코드를 작성하고, 파일을 만들고, 설정을 변경해줬습니다.

---

### 실제로 만들어진 기능들

#### 1. 업무달력
공무원이 매월 처리해야 할 업무를 한눈에 볼 수 있는 달력입니다. 오늘 할 일, 이번 주 할 일, 이번 달 할 일을 구분해서 보여줍니다. 사용자가 자기만의 업무를 추가할 수도 있습니다.

이 달력 하나의 코드가 **2,518줄**입니다. 제가 직접 쓴 건 아마 10줄도 안 될 겁니다.

#### 2. AI 업무가이드
업무를 클릭하면 AI가 자동으로 **업무 처리 절차**를 만들어줍니다. "원천세 신고·납부"를 클릭하면, 어떤 순서로 처리해야 하는지, 주의할 점은 뭔지, 관련 법령은 뭔지 AI가 분석해서 알려줍니다.

#### 3. 계약 실무 도구
- 수의계약/경쟁입찰 판단 도구
- 예정가격 계산기
- 설계변경 가이드
- 계약서 검토 도구
- 견적서 자동 분석

#### 4. 문서 분석
계약서나 내역서를 사진으로 찍어 올리면, AI가 자동으로 내용을 분석해주는 기능까지 만들었습니다.

---

### 삽질의 연속이었습니다

순탄하지만은 않았습니다. 솔직하게 적어봅니다.

**"가이드를 생성하는 중 오류가 발생했습니다"**

AI 업무가이드 기능을 만들고 나서 처음 클릭했을 때 나온 메시지입니다. 원인은 단순했습니다. API 키(AI 서비스 이용 열쇠)를 서버에 등록하지 않은 겁니다.

그런데 저는 "API 키"가 뭔지도 몰랐습니다.

클로드 코드에게 물어봤습니다:

> "가이드가 생성되지 않아."

AI가 서버 로그를 확인하더니 원인을 찾아줬습니다:

> "크레딧(잔액)이 부족합니다."

API 서비스에 가입은 했는데, 결제를 안 한 거였습니다. AI가 결제 화면까지 안내해줬고, $5를 충전하니 바로 작동했습니다.

**서버 배포도 난관이었습니다.**

"서버에 배포해줘"라고 했더니 Docker라는 프로그램이 필요하다고 합니다. Docker를 설치하고, 서버를 구매하고(Vultr라는 클라우드 서비스), SSH 키를 설정하고... 이 모든 과정을 AI와 대화하면서 하나씩 해결했습니다.

"Configure 버튼이 안 눌려" 같은 기초적인 질문도 했습니다. AI는 "운영체제를 선택하지 않아서 그렇다"고 정확히 알려줬습니다.

---

### 비전공자가 느낀 솔직한 후기

#### 좋았던 점

**1. 한국어로 말하면 된다**
영어 코드를 읽을 필요가 없습니다. "이런 기능 만들어줘"라고 한국어로 말하면 AI가 알아서 합니다.

**2. 실수해도 된다**
뭔가 잘못되면 AI에게 "안 돼"라고만 하면 됩니다. AI가 원인을 찾고 고쳐줍니다.

**3. 모르는 것을 물어볼 수 있다**
"Docker가 뭐야?", "API 키가 뭐야?" 같은 질문에도 쉽게 설명해줍니다. 누구한테 물어보기 부끄러운 질문도 AI에게는 편하게 할 수 있습니다.

**4. 24시간 일한다**
새벽에 작업해도, 주말에 해도 AI는 항상 똑같은 수준으로 도와줍니다.

#### 아쉬웠던 점

**1. 완전한 자동은 아니다**
AI가 만들어준 것이 바로 완벽하게 작동하는 경우는 드뭅니다. "이 부분 수정해줘", "이건 아니야, 이렇게 바꿔줘" 하는 피드백을 계속 주어야 합니다.

**2. 기본 개념은 알아야 소통이 된다**
서버가 뭔지, 데이터베이스가 뭔지 정도는 알아야 AI에게 정확하게 지시할 수 있습니다. 완전한 제로 지식으로는 어렵습니다. 다만 이 "기본 개념"도 AI에게 물어보며 배울 수 있습니다.

**3. 비용이 든다**
클로드 코드 자체 이용료, AI API 사용료, 서버 비용 등이 있습니다. 무료는 아닙니다.
