## 규칙
**도구 권위자 검증 / 법령 정정 / 시드 수정** 3개 컨텍스트에서는 메모리 권고를 즉시 적용하지 말고, mcp(법제처) 또는 PDF fallback으로 현행 조문/금액/유효성을 사전 검증한 후 적용한다.

**Why**: 2026-04-30 도구 권위자 검증 H1~H10 진행 중 메모리 권고 부정확이 6건 누적 발견됨 (낙찰하한율, 지체상금 한도, 폐지 조문 §42의2, 조문번호 전부개정, 면제기준, 적격심사 적용범위). 메모리는 "당시 진실" 스냅샷이며, 법령은 시간 경과로 stale 가능성이 디폴트. H10에서 §42의2가 2022년 삭제된 폐지 조문임을 mcp 검증으로 발견한 사례가 결정적 입증.

**How to apply**:
- 컨텍스트가 "도구 검증/법령 정정/시드 수정"이면 → 메모리 권고를 가설로만 채택 → mcp 검증 후 commit
- 그 외 컨텍스트(기획·전략·일반 대화)는 메모리 권고를 그대로 신뢰 (over-correction 방지)
- mcp 미발급 환경에서는 PDF 원문 인용으로 fallback, retroactive 검증 부채로 등록
- "메모리 → mcp 게이트 → 정정" 2단 파이프라인을 표준 운영 모델로 채택

## 5단계 게이트 (도구 권위자 검증 표준)
1. BLV (Before-Live-Validation): 라이브 결과화면 인용 조문 추출
2. lint: 조문 정합성 정적 분석
3. mcp 게이트 (의무): 법제처 현행 조문 검증
4. 라이브 회귀 검증: 정정 후 결과화면 재확인
5. retroactive 부채 등록: 연결 토픽·시리즈·플로차트 추후 일괄 검증
