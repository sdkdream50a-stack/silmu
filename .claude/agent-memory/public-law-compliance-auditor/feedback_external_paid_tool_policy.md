# 외부 유료 도구 도입 시 저작권 판단 프레임

## 핵심 원칙

silmu는 외부 유료 도구의 수치를 동일하게 인용해도 그 수치가 **법령 사실**이면 저작권 위반이 아니다. 외부 도구는 단지 "silmu의 부정확성을 깨닫게 한 trigger" 역할일 뿐이며, 정정 자료는 항상 법제처 원문에서 독립적으로 재확인해야 한다.

**Why**: 저작권법 §7 (보호받지 못하는 저작물) — 헌법·법률·명령·조약·조례·규칙·법원 판결·고시 등은 저작권 보호 대상에서 제외. 공무원연금법 부칙의 단계적 인하율 같은 법령 사실은 누가 시트에 옮겨 적든 그에게 권리가 발생하지 않는다. 또한 단순 사실의 나열(facts)은 창작성 결여로 §4 보호도 받지 못한다.

**How to apply**:

### 1단계: 도입 항목을 4범주로 분류

| 범주 | 저작권 위험 | 처리 |
|------|-----------|------|
| 법령 사실 (조문·부칙·고시 수치) | 없음 (§7) | 법제처 mcp로 독립 확인 후 인용 |
| 알고리즘·산식 로직 (idea) | 없음 (idea-expression dichotomy) | 학습 후 자체 재구현 가능 |
| 표현물 (셀 좌표·VBA 코드·UI 디자인·시트 레이아웃) | 있음 | 절대 복제 금지 |
| 창작적 해설문·예시 시나리오 | 있음 | 인용 시 출처 표기 또는 재작성 |

### 2단계: trigger vs 도입 구분

- **trigger(발견 계기)**: 외부 도구가 silmu의 오류를 알려줬다 → 저작권 무관
- **도입(복제·번안)**: 외부 도구의 표현물을 silmu에 가져왔다 → 저작권 검토 필요

**판단 기준**: silmu 정정 자료가 외부 도구 없이도 법제처 원문만으로 독립 작성 가능한가? Yes → trigger / No → 도입

### 3단계: 페르소나 적합성 판단

저작권상 합법이라도 다음 경우 도입 비권고:
- silmu 페르소나(일반 공무원·행정실 실무자)와 어긋남 (예: 정밀 시뮬레이션은 인사혁신처·연금공단 영역)
- 매년 개정 추적 부담으로 정확도 리스크 발생
- 신뢰 포지셔닝(기준·근거 안내) 역행
